# Codespace Response Rule

For every future repository or website change request, ChatGPT must include the exact Codespace terminal commands the user should run next.

Each response should include:

- the command to get the latest repository code into Codespace
- the command to verify the change
- the command to run the Codespace website preview
- the expected result
- the save/report command when the user wants the local change committed and documented

This instruction exists because the user relies on Codespace as the active website preview and needs every repo change to include terminal instructions that keep GitHub, Codespace, and the website preview aligned.
